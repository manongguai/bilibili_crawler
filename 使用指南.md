# 🎬 B站视频爬虫 - 超级简单使用指南

## ⚡ 一键运行

### 方法1：使用启动脚本（推荐）

**Mac/Linux 用户：**
```bash
./start.sh
```

**Windows 用户：**
```cmd
start.bat
```

### 方法2：直接使用Python

```bash
# 标准版本（速度快，可能遇到限制）
python run.py

# 智能版本（速度慢，成功率高）
python run.py smart
```

## 🎯 使用方法

### 交互式使用（最简单）
1. 运行程序后选择版本：
   ```
   请选择版本 (1/2) [默认:1]:
   ```
   - 1: 标准版本（快速）
   - 2: 智能版本（慢但稳定）

2. 输入你要爬取的用户UID，例如：`435776729`
3. 按回车，程序会自动爬取并保存数据

### 直接指定UID
```bash
# 标准版本
python run.py 435776729

# 智能版本（推荐遇到问题时使用）
python run.py smart 435776729
```

### 🔥 新功能：智能版本

专门针对B站反爬虫机制优化的版本：

**特点：**
- 🧠 智能延迟算法：根据失败次数自动调整等待时间
- 🎭 多User-Agent轮换：模拟不同浏览器
- 📊 分页控制：每页只获取5个视频，减少压力
- ⏰ 强制休息：每3页后休息60秒
- 🔄 自动重试：最多10次重试，递增延迟

**适用场景：**
- 标准版本遇到"请求过于频繁"错误时
- 爬取大量视频（>100个）时
- 网络环境不稳定时

## 📁 输出文件

爬取完成后，结果会保存在 `output` 文件夹中，文件名格式：
`videos_UID_时间戳.json`

例如：`videos_435776729_20251208_154500.json`

## 🔍 如何获取UID

1. 打开B站，进入目标用户的个人主页
2. 查看浏览器地址栏，找到类似这样的URL：
   ```
   https://space.bilibili.com/435776729
   ```
3. 数字部分就是UID（例如：`435776729`）

## ⚠️ 注意事项

1. **频率限制**：如果提示"请求过于频繁"，请等待几分钟再试
2. **仅爬公开数据**：只能爬取用户公开的视频
3. **网络环境**：确保网络连接正常
4. **学习用途**：仅用于学习和研究目的

## 🆘 常见问题

**Q: 程序报错怎么办？**
A: 大部分是网络问题，稍等片刻再试

**Q: 提示"用户不存在"怎么办？**
A: 检查UID是否输入正确，或者用户是否设置了隐私保护

**Q: 爬取到的数据在哪里？**
A: 在 `output` 文件夹里，JSON格式，可以用文本编辑器打开

**Q: 可以批量爬取多个用户吗？**
A: 可以，爬完一个用户后，程序会问是否继续爬取其他用户
